# 像素下樓梯:關卡主題重設計 — 設計文件

日期:2026-07-17
狀態:待郭哲宏審閱通過後才進入實作規劃(明確指示:先不要開始實作)

## 一、目標

把單一洞穴場景的下樓梯遊戲,改造成「多主題關卡循環」的版本,讓遊戲更耐玩:
- 關卡隨深度切換主題(叢林 → 賽博龐克),障礙物、背景、音效氛圍跟著換。
- 美術全面升級:改用 OpenGameArt 的專業素材包 + 程式疊加光影特效。
- 架構上做成「主題設定表」,之後加第三關(例如岩漿地獄)只要加一筆設定。

## 二、關卡結構

- 每關長度 100 公尺:叢林 0–100m → 賽博龐克 100–200m → 叢林第二輪 200–300m → …無限循環。
- 每進入新一輪:障礙物出現機率提高、捲動速度基準加快(各有封頂,保持可玩)。
- 換關瞬間:
  - 畫面橫幅提示(叢林:「🌿 幽暗雨林」/ 科幻:「⚡ 霓虹都市」,文案實作時可再調)。
  - 背景與牆面在幾層平台的距離內漸變換色。
  - 換關前後各留 2–3 層純普通平台當緩衝。

## 三、共通系統(不變的部分)

- 操作:← → / A D / 觸控左右半邊。
- 血量 10 格;愛心補血(+2),只長在安全平台上,越深機率越高。
- 受傷 +1 秒無敵閃爍;掉出畫面底部或血量歸零 = 遊戲結束。
- 深度 = 分數(28px = 1m);最高紀錄存 localStorage(key `downstairs-best` 沿用)。
- 尖刺天花板保留,但外觀隨主題換:叢林 = 荊棘樹刺、科幻 = 雷射發射列。
- 固定步長 120Hz 物理、360×640 邏輯解析度、WebAudio 合成音效,全部沿用。
- 主角:改用 Warped City 的賽博龐克主角(CC0),動畫:待機 / 跑 / 跳(下落)/ 受傷。原像素小人退役。

## 六、美術風格總則

## 八、技術做法

- 維持單一 `index.html`:所有用到的圖裁切後轉 base64 內嵌,不依賴外部檔案(Artifact 的 CSP 也要求如此)。
- 預估檔案大小:內嵌後約 1MB 上下,可接受;只嵌實際用到的圖格,不整包內嵌。
- 程式結構:新增 `THEMES` 主題設定表(調色盤、背景層、牆面畫法、障礙物種類與機率、動態元件),核心邏輯讀表運作;加新關卡 = 加一筆設定。
- 障礙物行為參數(張嘴週期、雷射開關時距、無人機速度等)集中在主題表,方便平衡調整。
- 音效:沿用 WebAudio 合成,依主題微調(科幻關加電子感音色)。

## 十、風險與備註

- 素材裁切工作量:fgtiles 素材表需要人工標座標裁格,是實作階段最花時間的部分。
- 舊紀錄相容:計分規則不變,舊的最高紀錄沿用。
- 平衡調整:五種科幻障礙 vs 四種叢林障礙,實測後可能需要調整出現機率讓兩關難度接近。
