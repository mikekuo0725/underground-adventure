## 一、目標

把單一洞穴場景的無限下樓梯遊戲,改造成「五大關 + 全破結局」的完整版:
- 五關各有主題,障礙物、背景、音效氛圍跟著換;過完第五關 = 全破,有結算畫面。
- 新增分數制:深度分 + 金幣道具分 + 全破獎勵,取代原本單純的深度計分。
- 美術全面升級:改用 OpenGameArt 的專業素材包 + 程式疊加光影特效。
- 架構上做成「主題設定表」,加關卡 = 加一筆設定。

### 難度曲線(2026-07-18 新增,參考同類遊戲設計)

參考來源:Downwell 的四區段節奏分析(每區逐步移除「安全選項」、新機制先單獨登場)、經典「鋸齒難度曲線」(difficulty saw:關內爬升、新機制登場時微降)、NS-Shaft(緩慢連續加速)、Half-Life(依玩家狀態動態擺放補給)。套用成以下規則:

1. **鋸齒型總曲線**:難度關內線性爬升,換關時回落一小段再繼續升 — 每關起始速度約為前關結尾的 93%,配合換關前後的 2–3 層純普通平台緩衝,形成「高峰 → 喘息 → 更高峰」的節奏。
2. **新障礙隔離教學**(參考 Downwell / 鋸齒曲線):每關前 30 公尺是教學段,該關的新障礙一種一種輪流登場(例如都市關:先只有玻璃雨遮,再加入冷氣機……),30 公尺後才混合出現。玩家永遠先單獨認識新東西,再面對組合技。
3. **關尾小高峰**:每關最後 15 公尺,障礙機率拉到該關上限(仍有封頂),過關瞬間釋放壓力 — 強化「撐過去就過關了」的成就感。
4. **速度與障礙參數表**(28px = 1m;實測後可微調,曲線形狀不變):

| 關卡 | 捲動速度(起 → 終,px/s) | 危險平台總機率(起 → 終) | 備註 |
|---|---|---|---|
| 1 晴空大樓 | 55 → 95 | 8% → 18% | 新手關,教學段最長 |
| 2 幽暗雨林 | 88 → 135 | 16% → 26% | |
| 3 霓虹都市 | 125 → 175 | 24% → 34% | 障礙種類最多的一關 |
| 4 冰封礦坑 | 163 → 210 | 30% → 40% | 全關滑冰慣性,教學段先讓玩家適應滑行 |
| 5 熔火地心(最終關) | 195 → 245 | 36% → 46% | 全關平台融化,見下條「最終關絕境感」 |

5. **最終關絕境感**(參考 Downwell 第四區「reductionist」手法):第五關把普通平台比例壓到最低、愛心幾乎不再出現,玩家被迫用前四關練成的所有技巧不斷下降 — 全破的爽感來自「絕境中殺出來」。
6. **隱性動態輔助**(參考 Half-Life):血量 ≤ 4 格時,愛心出現機率悄悄提高 50%(不告知玩家);血量健康時恢復正常。避免新手在中段卡死,又不影響高手挑戰滿血全破。
7. **公平性原則**:所有會扣血的障礙一律有預告(冷氣風扇加速、食人花抖動、電擊板黃燈、雷射固定節奏),死掉永遠是「我自己沒躲好」而不是「遊戲陰我」;危險平台不連續兩層出現的規則沿用到所有關卡。
- 換關瞬間:
  - 畫面橫幅提示(都市:「☀️ 晴空大樓」/ 叢林:「🌿 幽暗雨林」/ 科幻:「⚡ 霓虹都市」/ 冰礦:「❄️ 冰封礦坑」/ 熔火:「🔥 熔火地心」,文案實作時可再調)。
  - 背景與牆面在幾層平台的距離內漸變換色。
  - 換關前後各留 2–3 層純普通平台當緩衝。
- 全破:過完第五關先落到「遠古祭壇」彩蛋場景(見第八章),停 2–3 秒後播放結算畫面(總分、深度、金幣收集數、剩餘血量獎勵),可重玩挑戰高分。
- 死亡(血量歸零或掉出畫面):遊戲結束畫面同樣顯示分數結算。

## 三、共通系統(不變的部分)

- 操作:← → / A D / 觸控左右半邊。
- 血量 10 格;愛心補血(+2),只長在安全平台上,越深機率越高。

### 分數制(新)
- 總分 = 深度分 + 金幣分 + 全破獎勵:
  - 深度分:每下降 1 公尺 +10 分(28px = 1m,沿用)。
  - 金幣:+100 分;大金幣:+500 分(稀有,放在較難拿的位置,例如尖刺平台旁、蜂群巡邏路線上)。
  - 全破獎勵:剩餘血量每格 +500 分(滿血全破多 5000 分)。
- 金幣造型隨關卡變,玩法一致(碰到即得分 + 音效):都市 = 金幣、叢林 = 金色果實、賽博 = 發光晶片、冰礦 = 發光冰晶(大金幣 = 大冰晶簇)、熔火 = 遠古金幣(帶符文;大金幣 = 熔金塊)。
- 金幣出現位置:平台上方、平台之間的下落路徑上(引導玩家路線),偶爾排成小串。
- 最高紀錄:localStorage 改存兩項 — 最高總分(新 key `downstairs-best-score`)與最深深度(沿用 `downstairs-best`,全破後顯示 500m)。
- 尖刺天花板保留,但外觀隨主題換:都市 = 施工鷹架鋼筋尖端、叢林 = 荊棘樹刺、科幻 = 雷射發射列、冰礦 = 倒插冰柱列、熔火 = 滴著岩漿的黑曜石鐘乳石列。
- 固定步長 120Hz 物理、360×640 邏輯解析度、WebAudio 合成音效,全部沿用。
- 主角(2026-07-19 改,同日造型定案):原創「探險少年」— 棕髮、額上推著護目鏡、紅圍巾飄動、小背包;程式繪製像素圖,動畫:待機 / 跑 / 落下 / 受傷。原像素小人與原定的 Warped City 主角都退役(Warped City 素材仍用於第三關環境、無人機、飛行車)。
- 造型已過目定案:**A・經典探險裝**(22×30 像素、輪廓線 + 雙層陰影;卡其探險衣、金框護目鏡天藍鏡片、鮮紅圍巾、皮帶靴)。定案像素圖與色盤存於 `assets/character/adventurer-boy-drafts.html`(選 A 款),站姿正面為造型基準;跑步 / 落下(圍巾上飄)/ 受傷姿勢實作時衍生,**完成後同樣給郭哲宏過目**。質感基準:參考 Jetpack Joyride 的細節密度(不抄造型),像素風呈現。

### 場景概念
- 賽博城市的地基之下,是一座被冰封的廢棄礦坑:暗色岩壁、結冰的木架與軌道、發光藍冰晶是主光源。
- 視覺節奏接在霓虹炫亮之後轉入「幽藍冷冽」,延續越掉越奇幻的軸線。

### 核心手感:全關滑冰慣性
- 所有平台都結冰,帶中等強度慣性:落地後繼續滑行、煞車要提前,整關操作手感煥然一新。
- 唯一例外是減速雪堆(見障礙表):踩進去移動速度減半、完全不滑,是威脅也是煞車工具。
- 前 30m 教學段障礙極少,讓玩家先適應滑行;實測若過難,調降慣性強度(參數集中在主題表)。

### 環境素材
- 主素材包:OPP2017 – Cave and Mine Cart(Open Pixel Project / Hapiel 等 35 位畫師,CC0)。已下載解壓:`assets/ice/opp_cave/`。
- 用途:兩側洞壁與普通平台(自帶積雪邊緣)、藍色冰晶裝飾、礦車(直軌 + 斜坡動畫現成)與軌道磚、背景岩壁(bg_cave、bg_crystal、bg_fluo 可作視差層)。
- 蝙蝠:Bat sprite(bagzie,OGA-BY 3.0,需署名)。已下載:`assets/ice/bat-sprite.png`;實作時調偏藍加強冰感。

### 背景動態(全部採用)
1. 多層視差捲動(近景冰壁 → 礦坑支架與軌道殘骸 → 深處發光冰晶洞窟)。
2. 發光冰晶明滅:牆面藍冰晶緩慢呼吸般明暗,整關主光源。
3. 飄落雪粉與冰晶粒子:從頭頂裂縫灑下,細碎閃光。
4. 廢棄提燈搖晃:背景提燈輕擺、偶爾閃爍,暖橘光點綴冷色畫面。
5. 主角呼出白霧:角色定期哈出一小口白氣,強化寒冷體感。
6. 彩蛋:背景冰層裡隱約凍著第三關的巡邏無人機與舊礦工機器人,暗示「賽博城市的東西掉下來被冰封了」。

## 八、熔火地心關(🔥)— 第五關(最終關)

### 場景概念
- 冰封礦坑再往下,冰層融盡、岩壁轉為黑曜岩,整個畫面被岩漿的橘紅光照亮 — 與第四關的幽藍冷冽形成整趟旅程最強的色彩對比。
- 背景深處隱約可見**半融化的遠古石柱與符文雕刻**:這裡曾有文明,也為底部的寶藏鋪路(場景方向「熔岩中的遠古痕跡」)。

### 核心手感:一切都在融化
- **所有平台踩上即開始融化**:發紅 → 邊緣滴落 → 約 1.5 秒後沉入消失。不能久站、不能猶豫,只能一路往下。
- 唯一例外:**遠古石板平台**(遺跡殘片,耐熱)融化較慢,約 2.5 秒 — 是這關唯一喘息點,也呼應場景故事。
- 教學段(前 30m)平台融化速度放慢一半,讓玩家先體會規則再加壓。
- 與第四關互為鏡像:冰關是「停不下來」(滑),熔火關是「不准停」(融);前四關練的急停急煞技巧在此全部用上。

## 十二、技術做法

- 前端維持單一 `index.html`:所有用到的圖裁切後轉 base64 內嵌,不依賴外部圖檔。
- 後端(2026-07-19 新增):一個 Cloudflare Worker 同時負責送遊戲網頁與排行榜 API(`GET /api/scores` 前 10、`POST /api/scores` 送分),分數存 D1;前端只多呼叫這兩支 API。
- 預估檔案大小:內嵌後約 1.5MB 上下(四關,第五關再往上加),可接受;只嵌實際用到的圖格,不整包內嵌。
- 程式結構:新增 `THEMES` 主題設定表(調色盤、背景層、牆面畫法、障礙物種類與機率、動態元件),核心邏輯讀表運作;加新關卡 = 加一筆設定。
- 障礙物行為參數(張嘴週期、雷射開關時距、無人機速度等)集中在主題表,方便平衡調整。
- 音效:沿用 WebAudio 合成,依主題微調(科幻關加電子感音色);BGM 為 CC0 音樂檔,與遊戲網頁一起放 Cloudflare 靜態檔案(不內嵌 base64),詳見第十章。

## 十四、風險與備註

- 素材裁切工作量:fgtiles 素材表需要人工標座標裁格,是實作階段最花時間的部分。
- 舊紀錄相容:計分規則改版,最深深度紀錄(`downstairs-best`)沿用;總分是新制,從零開始累積(新 key)。
- 平衡調整:都市 4 種 / 叢林 4 種 / 科幻 5 種 / 冰礦 5 種 / 熔火 5 種障礙,實測後需調整出現機率,讓難度逐關上升、第五關最難但仍可全破。
- 滑冰慣性風險:冰礦關全關慣性是手感大改,新手可能不適應;教學段放緩 + 慣性強度做成可調參數,實測後平衡。
- 融化平台風險:第五關全關平台限時消失,若 1.5 秒實測太苛(尤其配上最高捲動速度),融化秒數與教學段減速倍率做成可調參數,實測後平衡。
- 金幣擺放:位置生成規則要跟平台生成整合(避免出現在牆裡或拿不到的地方),實作時需實測調整。
- 主角草案與封面插畫:兩者都要先給郭哲宏過目才算定案(照既有美術過目慣例);封面插畫是新增的較大美術工作量,實作時程要預留。
- 排行榜後端是全新工作量(Worker + D1 + API + 三字母輸入 UI);防灌水只做簡易版,擋得住路人但擋不住有心人,個人遊戲可接受。
- 換部署平台:網址會變,舊 Artifact 連結的朋友要重新分享;D1/Workers 免費額度對小遊戲綽綽有餘。
- 內嵌素材增加(五關背景 + 元件),檔案預估上修到 1.5MB 上下,仍可接受;維持只嵌實際用到的圖格;BGM 音樂檔另放靜態檔案不計入。
- BGM 選曲工作量:7 段 CC0 chiptune 要逐首驗證授權 + 試聽頁過目;OpenGameArt 搜尋頁摘要有幻覺前科,一律進詳細頁驗證;若個別段落找不到合適曲目,退回程式合成補位(第十章)。
