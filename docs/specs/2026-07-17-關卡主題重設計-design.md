# 像素下樓梯:關卡主題重設計 — 設計文件

日期:2026-07-17(2026-07-18 修訂:新增現代都市關為第一關,關卡順序改為 都市 → 叢林 → 賽博龐克)
狀態:待郭哲宏審閱通過後才進入實作規劃(明確指示:先不要開始實作)

## 一、目標

把單一洞穴場景的下樓梯遊戲,改造成「多主題關卡循環」的版本,讓遊戲更耐玩:
- 關卡隨深度切換主題(現代都市 → 叢林 → 賽博龐克),障礙物、背景、音效氛圍跟著換。
- 美術全面升級:改用 OpenGameArt 的專業素材包 + 程式疊加光影特效。
- 架構上做成「主題設定表」,之後加第四關(例如岩漿地獄)只要加一筆設定。

## 二、關卡結構

- 每關長度 100 公尺:現代都市 0–100m → 叢林 100–200m → 賽博龐克 200–300m → 現代都市第二輪 300–400m → …無限循環。
- 三關視覺節奏:白天晴朗 → 月夜幽暗 → 霓虹炫亮,從亮到暗再到炫,「越掉越深、從日常掉進奇幻」。
- 每進入新一輪:障礙物出現機率提高、捲動速度基準加快(各有封頂,保持可玩)。
- 換關瞬間:
  - 畫面橫幅提示(都市:「☀️ 晴空大樓」/ 叢林:「🌿 幽暗雨林」/ 科幻:「⚡ 霓虹都市」,文案實作時可再調)。
  - 背景與牆面在幾層平台的距離內漸變換色。
  - 換關前後各留 2–3 層純普通平台當緩衝。

## 三、共通系統(不變的部分)

- 操作:← → / A D / 觸控左右半邊。
- 血量 10 格;愛心補血(+2),只長在安全平台上,越深機率越高。
- 受傷 +1 秒無敵閃爍;掉出畫面底部或血量歸零 = 遊戲結束。
- 深度 = 分數(28px = 1m);最高紀錄存 localStorage(key `downstairs-best` 沿用)。
- 尖刺天花板保留,但外觀隨主題換:都市 = 施工鷹架鋼筋尖端、叢林 = 荊棘樹刺、科幻 = 雷射發射列。
- 固定步長 120Hz 物理、360×640 邏輯解析度、WebAudio 合成音效,全部沿用。
- 主角:改用 Warped City 的賽博龐克主角(CC0),動畫:待機 / 跑 / 跳(下落)/ 受傷。原像素小人退役。

## 四、現代都市關(晴空大樓)— 第一關

### 場景概念
- 從摩天大樓頂樓沿外牆一路往下:兩側牆面是大樓玻璃帷幕與磚牆立面,平台是窗台、雨遮、外牆突出物。
- 白天晴朗:藍天白雲、遠景雪山與粉彩色城市天際線。
- 定位是新手關:障礙物全部有明確預告,行為最單純。

### 環境素材
- 主素材包:City Mega Pack(GrafxKid,CC0,作者希望署名 GrafxKid)。已下載:`assets/city/CITY_MEGA.png`、`assets/city/SHIPPING_DOCKS_AREA.png`。
- 背景:素材包內建白天城市背景(藍天雲層 + 雪山 + 粉彩天際線),切成多層做視差。
- 牆面 / 普通平台:素材包的建築立面磚(磚牆、窗戶、玻璃帷幕),直向拼接成兩側大樓外牆。
- 素材包沒有的元件(冷氣室外機、玻璃雨遮、鴿子等)由程式照 GrafxKid 色盤繪製,維持風格統一。
- 美術取捨(已定案):明亮卡通感與後兩關的暗色奇幻形成反差,強化「從日常掉進奇幻」的層次;再以共通的光影特效(光柱、光暈、暗角)拉齊三關質感。

### 背景動態(全部採用)
1. 多層視差捲動(天空雲層 → 雪山天際線 → 遠景大樓)。
2. 雲朵多層飄動;飛機拉凝結尾橫越,偶爾直升機懸停。
3. 玻璃帷幕隨機閃出陽光反光;斜射光柱(程式疊加,延續「精緻像素 + 真實光影」)。
4. 遠景鳥群排隊掠過;陽台晾衣隨風擺動;窗內偶爾人影走過。
5. 廣告飛船拉布條緩緩橫越,布條寫「像素下樓梯」當彩蛋。

### 障礙物(4 種,都是大樓外牆的日常物件)
| 障礙物 | 行為 | 素材 |
|---|---|---|
| 玻璃雨遮 | 踩上去裂痕擴大,約 0.8 秒後碎掉(不扣血;對應叢林斷裂樹枝) | 程式繪製(半透明玻璃 + 裂紋動畫) |
| 冷氣室外機 | 循環:安全 → 風扇加速警告 → 噴熱風;噴風時站上扣 2 血(對應食人花 / 電擊板) | 程式繪製(風扇轉動 + 熱風波紋) |
| 鴿子群 | 空中左右飛,碰到扣 2 血,被撞會撲翅散開(對應蜂群 / 無人機) | 程式繪製(2–3 格拍翅動畫) |
| 帆布遮雨棚 | 條紋帆布,把玩家往上彈(對應彈跳蘑菇 / 彈簧) | 素材包遮雨棚元件 + 程式彈跳形變 |

- 尖刺天花板的都市版外觀:施工鷹架的鋼筋尖端。

## 七、美術風格總則

- 「精緻像素 + 真實光影」:像素素材為底,程式疊加平滑光影 — 漸層光柱、光暈、流動薄霧、畫面上下暗角。
- 兩關的環境素材都是 16px 系像素風,縮放繪製時維持 image-rendering: pixelated。

## 九、技術做法

- 維持單一 `index.html`:所有用到的圖裁切後轉 base64 內嵌,不依賴外部檔案(Artifact 的 CSP 也要求如此)。
- 預估檔案大小:內嵌後約 1.2MB 上下(三關),可接受;只嵌實際用到的圖格,不整包內嵌。
- 程式結構:新增 `THEMES` 主題設定表(調色盤、背景層、牆面畫法、障礙物種類與機率、動態元件),核心邏輯讀表運作;加新關卡 = 加一筆設定。
- 障礙物行為參數(張嘴週期、雷射開關時距、無人機速度等)集中在主題表,方便平衡調整。
- 音效:沿用 WebAudio 合成,依主題微調(科幻關加電子感音色)。

- 素材裁切工作量:fgtiles 素材表需要人工標座標裁格,是實作階段最花時間的部分。
